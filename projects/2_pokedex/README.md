---

## 📌 Sobre o Projeto

Este projeto é um exercício guiado da oficina, onde os participantes constroem uma **Pokédex interativa** consumindo a [PokéAPI](https://pokeapi.co/) — com toda a jornada de desenvolvimento **orientada por documentos Markdown gerados com IA**.

A ideia central é simular um fluxo real de desenvolvimento de software, passando por diferentes **papéis (roles)**, onde cada um produz um artefato que serve de entrada para o próximo.

---

## 🔄 Fluxo de Trabalho com IA

O projeto segue a metodologia **Spec Driven Development (SDD)**: cada decisão é documentada em um arquivo `.md` antes de ser implementada.

```
🏢 Negócio  →  🕵️ Analista  →  ⚙️ Engenheiro  →  💻 Desenvolvedor
```

| Role | Responsabilidade | Artefato Gerado |
|---|---|---|
| 🏢 **Negócio** | Define o objetivo e valor do projeto | `business.md` |
| 🕵️ **Analista** | Transforma o objetivo em requisitos claros | `spec.md` |
| ⚙️ **Engenheiro** | Planeja tecnicamente como construir | `tasks.md` |
| 💻 **Desenvolvedor** | Implementa seguindo as tasks | Código |

---

## 📂 Estrutura Esperada

```
pokedex/
 ├── README.md        # Este arquivo
 ├── business.md      # 🏢 Visão de negócio
 ├── spec.md          # 🕵️ Requisitos e critérios de aceite
 ├── tasks.md         # ⚙️ Tasks técnicas detalhadas
 └── index.html       # 💻 Implementação final
```

---

## 🚀 Como Participar

1. Peça à IA para criar cada documento no seu papel
2. Revise e ajuste antes de passar para o próximo
3. Use o `tasks.md` como guia para o desenvolvimento
4. Implemente com a ajuda da IA, seguindo as tasks

> 💡 **Lembre-se:** A IA *propõe*, o humano *aprova*.
